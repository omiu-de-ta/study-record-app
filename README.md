## 概要
学習内容をシンプルに記録・管理できるWebアプリです。  
記録の手間を減らし、継続しやすいことを目的として開発しました。
最小限の操作で記録できるよう設計し、クリックで編集できるなど、直感的に操作できるUIを意識しています。

----

## 設計思想
本アプリでは「継続しやすさ」を最優先に設計しました。
入力項目を最小限にし、学習内容を入力するだけで日付が自動記録される仕組みにしています。
これにより、記録の手間を減らし、継続しやすい設計としました。
また、データをオブジェクトとして管理し、状態と表示を分離することで、不具合を減らしつつ、今後の機能追加にも対応しやすい構造にしています。

----

## 作成した背景
学習記録を残すことで、復習しやすくなり、自分の成長を客観的に振り返ることができると感じました。
しかし、記録する手間が負担となり、継続できないという課題がありました。
そこで、最小限の操作で簡単に記録できるシンプルなアプリを開発しました。

----

## 機能
- 学習内容の追加ができます。
- 学習内容の一覧表示ができます。
- 学習内容の削除ができます。
- 学習内容の編集ができます（クリックで編集モードへ移行）。
- 学習日付が自動記録されます。
- localStorageによるデータ永続化に対応しています（リロード後も保持されます）。

----

## 使い方
1. 入力欄に学習内容を入力します。
2. 「追加」ボタンをクリックします。
3. 一覧に記録が表示されます。
4. 内容をクリックすると編集できます。
5. 「削除」ボタンで削除できます。

## 工夫した点
- クリック操作で編集できる直感的なUIを実装し、操作の分かりやすさを意識しました。
- 学習日付を自動記録することで、振り返りしやすい設計にしました。
- 入力項目を最小限にすることで、継続しやすいシンプルな操作性を実現しました。
- 空入力を防ぐバリデーションを追加し、不要なデータ登録を防止しました。

----

## 設計・実装のポイント
- render関数でUIを一括再描画することで、状態と表示の不整合を防止しています。
- editingIdで編集状態を管理し、追加処理と編集処理を分離しています。
- map / filterを使用し、イミュータブルなデータ更新を意識しています。
- localStorageを用いたデータの永続化を実装しています。
- CSSを分離し、ロジックと見た目の分離による保守性を向上させています。
- IDでデータを一意に管理し、削除処理の正確性を担保しています。
- save関数とrender関数を分けることで、データ更新と描画の責務を分離しています。

----

## 苦労した点
削除機能の実装時に、意図しないデータが削除される問題がありました。
原因はデータを一意に識別できていなかったことだったため、IDで管理する設計に変更して解決しました。
また、データの読み込みと配列の初期化の順序によって、保存データが上書きされてしまう問題もありました。
console.logでデータの流れを確認しながら、処理順序を見直すことで対応しました。
さらに、当初はrender関数内で保存処理も行っていたため、機能追加が難しい構造になっていました。
そのため、save関数と分離し、データの流れを明確にしました。

----

## 学んだこと
- JavaScriptで基本的なCRUD処理（追加・削除・編集）を実装できるようになりました。
- map / filterを使ったデータ操作に慣れました。
- localStorageを用いたデータ保存の基本を理解しました。
- まだ理解が浅い部分もあるが、DOM操作と再描画の流れを実装できました。
- 状態と表示を分離することで、不具合の原因特定がしやすくなり、保守性が向上することを学びました。
