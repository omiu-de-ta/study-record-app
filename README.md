## 概要
学習内容をシンプルに記録・管理できるWebアプリです。  
記録の手間を減らし、継続しやすいことを目的として開発しました。
クリックで編集できるなど、直感的に操作できるUIを意識しています。

----

## 設計思想
本アプリでは継続して使用できるということを年頭におき、機能を必要なものだけを選び作成しました。
ですので、学習内容を入力するだけで、自動で日付情報が追加されるように設計しました。
データをオブジェクトとして管理することで、操作の不具合を減らすよう努力しました

----

## 作成した背景
日々の学習の中で、記録を残すことで何を勉強したのかがという目的意識が明確になったり、復習しやすくなるなど、記録することがとても重要だと思いました。記録することで自分の成長を客観的に振り返ることができると感じました。  
しかし、記録する手間が負担となり、継続できないという課題がありました。  
そこで、最小限の操作で簡単に記録できるシンプルなアプリを作成しようと思いました。

----

## 機能
- 学習内容の追加
- 学習内容の一覧表示
- 学習内容の削除
- 学習内容の編集（クリックで編集モードへ）
- 学習日付の自動記録
- localStorageによるデータ永続化（リロード後も保持）

----

## 使い方
1. 入力欄に学習内容を入力
2. 「追加」ボタンをクリック
3. 一覧に記録が表示される
4. 内容をクリックすると編集可能
5. 「削除」ボタンで削除

----

## 使用技術

- JavaScript（Vanilla JS）

## 工夫した点
- クリック操作で編集できる直感的なUIを実装しました。
- 学習日付を自動記録し、振り返りしやすい設計にしました。
- 入力項目を最小限にし、継続しやすいシンプルな操作性を実現しました。
- 空入力を防ぐバリデーションを追加し、入力ミスを防止しました。

----

## 設計・実装のポイント
- render関数で更新ごとにUIを一括再描画する設計にすることで、状態と表示の不整合を防止できるようにしました。
- editingIdで編集状態を管理し、追加処理と編集処理を分けることで、可読性とメンテナンスの向上をさせました。
- map / filterを使用し、値の書き換えが起こりずらいようにすることで、バグを減らす設計にしました。
- localStorageからの読み込み・保存処理を実装し、データの永続化を実現しました。(リロード後もデータを維持)
- CSSを分離し、見た目とロジックの分離させ、今後コードの修正をしやすいようにしました。
- IDを用いてタスクを一意に管理し、削除処理の正確性を高めました。
- if (taskText.trim() === "") return;を使い、見た目上は入力されていても実質空文字になるケースを防ぎ、不要なデータ登録を防止していま　　す。
- save関数とrender関数を作成することで、データ保存→描画の流れを理解しやすくしました

----

## 苦労した点
配列でデータ管理をすると削除機能の実装時に、意図しないデータが削除される問題がありました。  
そこでIDで管理する設計に変更することで解決しました。
また、データを読み込んだ後に配列を宣言するとデータが上書きされて、データが消えることがありました。
データがどいった処理をされいるか流れを考えたり、デバック作業でconsole.logでデータを表示して確認しながら作業することで対応しました。
最初にrender関数の中でlocalStorageの保存をしてしまい、機能を追加していくことが難しかった。
ですので、render関数とsave関数にわけて、データの流れを視覚的に終えるようにした。

----

## 学んだこと
- JavaScriptで基本的なCRUD処理（追加・削除・編集）を実装できるようになりました。
- map / filterを使ったデータ操作に慣れてきました。
- localStorageを使ったデータ保存の基本を理解した。
- まだ理解が浅い部分もあるが、DOM操作と再描画の流れを実装できました。
- データを保存→再描画という一連の処理の流れを理解することができました。
